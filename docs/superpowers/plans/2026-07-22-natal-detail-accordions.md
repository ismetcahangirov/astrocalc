# Natal Chart Detail Accordions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make each row in the natal chart's "Chart details" section tappable so it expands, accordion-style, to reveal what that placement means — with only one row open at a time — and add the missing "house meaning" interpretation content so house rows have text too.

**Architecture:** The meaning text already comes from the backend `/interpretations/for-chart` endpoint (planet-sign, planet-house, aspect). We add a new `house` interpretation category (12 generic house meanings × 4 locales) generated by the same original-content template pipeline, wired through the service/route so `for-chart` also returns house readings. On mobile, `formatChartDetails` attaches each row's interpretation `subjectKey`, and `NatalChartScreen` renders the detail rows as single-open accordions that look up their meaning from the fetched interpretation, folding the old flat "Your Reading" list into the accordions.

**Tech Stack:** TypeScript monorepo — `@astrocalc/calc-engine` (pure logic + Vitest), `apps/backend` (Express + Drizzle + Vitest), `apps/mobile` (React Native + Expo + Vitest). Interpretation content is DB-stored, seeded from `seedContent.ts`, admin-editable.

---

## Decisions locked in

- **House content = generic per-house meaning** (`house-1` … `house-12`), one subject per house number, independent of the sign on the cusp. Reuses the existing `HOUSE_AREA` + `HOUSE_ORDINAL` phrase banks already in `seedContent.ts`. 12 subjects × 4 locales = 48 new rows.
- **New category name:** `house` (added to `InterpretationCategory`).
- **Accordion joins meaning by `subjectKey`.** `formatChartDetails` attaches the exact subjectKey(s) each row maps to; the screen builds one flat `Map<subjectKey, content>` from the fetched interpretation (subjectKeys don't collide across categories) and looks rows up.
- **Fold the reading in, remove the duplicate flat list.** The separate "Your Reading" and bottom "Aspects" reading sections are removed — their content now appears inside the matching accordion rows. This avoids showing every reading twice.
- **Planet rows show two paragraphs** when available: the planet-in-sign meaning, then the planet-in-house meaning.
- **Angle rows (Ascendant / Midheaven) are not expandable** — there is no interpretation content for them in scope. They render as plain, non-tappable rows exactly as today.
- **Offline / reading-fetch error:** rows stay tappable; an expanded row whose meaning is missing shows a short localized "reading needs internet" note (reusing the existing `interpretationError` state).

```ts
function composeHouse(house: number, locale: InterpretationLocale): string {
  const area = HOUSE_AREA[house]![locale];

  switch (locale) {
    case 'en':
      return `The ${HOUSE_ORDINAL[house]!.en} house governs ${area}. It shows the area of life where these themes play out for you.`;
    case 'az':
      return `${HOUSE_ORDINAL[house]!.az} ev ${area} sahəsini idarə edir. Bu, həmin mövzuların həyatınızda təzahür etdiyi sahəni göstərir.`;
    case 'tr':
      return `${HOUSE_ORDINAL[house]!.tr} ev ${area} alanını yönetir. Bu temaların yaşamınızda ortaya çıktığı alanı gösterir.`;
    case 'ru':
      return `Дом ${house} управляет сферой: ${area}. Он показывает область жизни, где эти темы проявляются для вас.`;
  }
}
```

```tsx
if (Platform.OS === 'android' && UIManager.setLayoutAnimationEnabledExperimental) {
  UIManager.setLayoutAnimationEnabledExperimental(true);
}

interface AccordionRowProps {
  name: string;
  value: string;
  /** Small tag rendered after the name, e.g. a retrograde "R". */
  tag?: ReactNode;
  expanded: boolean;
  onToggle: () => void;
  /** The meaning content revealed when expanded. */
  children: ReactNode;
}

/**
 * One tappable, single-open detail row for the natal-chart details list. The
 * parent owns which row is open (so only one is), toggling via {@link onToggle};
 * this component just animates its body in and out and shows a caret.
 */
export function AccordionRow({ name, value, tag, expanded, onToggle, children }: AccordionRowProps) {
  return (
    <View style={styles.wrap}>
      <Pressable
        accessibilityRole="button"
        accessibilityState={{ expanded }}
        onPress={() => {
          LayoutAnimation.configureNext(LayoutAnimation.Presets.easeInEaseOut);
          onToggle();
        }}
        style={styles.header}
      >
        <Text style={styles.name}>
          {name}
          {tag}
        </Text>
        <View style={styles.right}>
          <Text style={styles.value}>{value}</Text>
          <Text style={styles.caret}>{expanded ? '▲' : '▼'}</Text>
        </View>
      </Pressable>
      {expanded ? <View style={styles.body}>{children}</View> : null}
    </View>
  );
}
```
